Preparing for HW1: Tasks for appUser $username

All IT systems of MI-6 have been compromised. Her Majesty's appUsers are at risk.

Your task is to develop a new secret appUser management system for MI-6.

Don't work with anyone else on this. Don't share any information. You may only trust some of the TAs.

Task 1: Configuration

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes

Create class com.example.skillbuild.domain.AppUser. With the following private fields:

  1. int id
  2. String name
  3. String university

Generate necessary getters and setters.

Task 3: Controller (simple)

Create controller com.example.skillbuild.controller.MainController.

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists).

Task 4: Controller, View, and Form

Create field public static List<AppUser> appUsers in class com.example.skillbuild.skillsBuildApplication. Initialize the list to contain two appUsers.

Create controller com.example.skillbuild.controller.AppUserController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/appUsers" should show view appUsers/list.
    • Add the list of all appUsers from com.example.skillbuild.skillsBuildApplication to the model as attribute appUsers.
    • Create the corresponding JSP to list the name and university of each appUser followed by a link to URL "/terminate?appUser={id}", e.g., "/terminate?appUser=3".
  2. Requests for URL "/newAppUser" should show view appUsers/form. Create the corresponding JSP with a form with modelAttribute="appUser" and action="/addAppUser" and inputs for all fields of class AppUser.
  3. POST requests for URL "/addAppUser" should add a new appUser to the list of all appUsers in com.example.skillbuild.skillsBuildApplication and redirect to /appUsers.

Task 5: GET Requests

Extend controller com.example.skillbuild.controller.AppUserController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/terminate" with request parameter appUser={id} should delete the appUser with the corresponding ID from the list of all appUsers in com.example.skillbuild.skillsBuildApplication and redirect to /appUsers.

Task 6: Validation

Set up a validator for class AppUser and make it available in controller AppUserController.

When adding a new appUser validate that

  1. The value of field id is unique within the list LabExtraApplication.appUsers.
  2. The value of field name is not empty.
  3. The value of field university is different from "UK" (MI-6 deploy appUsers only in foreign countries).
In case of validation errors go back to the form and show error messages for each field in the form.
